## ⚠️ Launch-blocker flagged by UX (not in original SEO scope)

**CLI naming drift: `reveren run` vs `rvr` is inconsistent across site copy.**

Per project memory ([`feedback_reveren_cli_naming.md`](../../../../../.claude/projects/-Users-innocentmuisha-Software-mrktable/memory/feedback_reveren_cli_naming.md)): *"reveren is the brand; `rvr` is the typed CLI binary (npm-style). Don't collapse them."*

But the site copy uses `reveren run` everywhere:
- `components/Terminal/Terminal.frames.ts:26` — types `$ reveren run plan-engineering` in the hero animation
- `app/manifesto/copy.ts:58` — body copy says `Try \`reveren run plan-engineering\``
- `components/PricingHeader/PricingHeader.tsx:17` — `Local <code>reveren run</code>`
- `lib/install.ts` + `lib/install.test.ts:10-12` — install test *guards against* `rvr` ever appearing, asserts `INSTALL_COMMAND` does not match `/\brvr\b/`

**Two interpretations, owner must pick one:**
1. **Memory is correct, site is wrong.** Rename across `Terminal.frames.ts`, `PricingHeader.tsx`, `pricing.ts`, `pricingFootnote`, `faq.ts`, `manifesto/copy.ts`. Update the guard test to assert `rvr`, not `reveren`.
2. **Site is correct, memory is stale.** Update [`feedback_reveren_cli_naming.md`](../../../../../.claude/projects/-Users-innocentmuisha-Software-mrktable/memory/feedback_reveren_cli_naming.md) to drop the `rvr` claim.

**Cost of inconsistency:** an engineer copies `reveren run plan-engineering` from the hero animation, gets `command not found`, walks away. The hero animation is the entire above-the-fold conversion surface — this is unrecoverable.

## Recommended order of implementation

Ranked by leverage. Owner decides what ships.

1. **Resolve the `reveren run` vs `rvr` naming once, then sweep + lock with a test.** Launch-blocker. See top of doc.
2. **Add `metadata.alternates.canonical = "/"` and per-page `description` to `app/page.tsx`.** One-line fix. Closes Critical C1. Removes the canonical-chain break that fragments brand-search SERP.
3. **Ship gate-aware `app/sitemap.ts` + `app/robots.ts`.** When `LAUNCH_MODE=coming-soon`, sitemap lists only `/`; robots disallows the `/coming-soon` rewrite from being treated as canonical for other routes. When the gate flips, sitemap auto-expands. Same files, two states. 4. **Rasterize the OG image to PNG 1200×630 + set per-route `openGraph` blocks.** Highest-leverage UX×SEO win on this site. Slack/LinkedIn don't render SVG OG; Twitter/Facebook/Discord don't either. Closes High H1, H2, L1.
5. **Add `description` to `/security`, `/privacy`, `/terms`, `/dpa`.** Trust surface — Googling "reveren security" currently returns generic marketing copy. Lift one sentence from each page's body. Closes High H3.
6. **Per-page JSON-LD on `/pods`, legal pages, `WebPage` on policy routes.** Closes Medium M1, M2.
7. **`FeatureTeaser` variant fix** — add `variant="h3"` so rendered class is `MuiTypography-h3` not `MuiTypography-body1`. Closes Medium M4.

Items 1–4 are the launch-readiness checklist. Items 5–7 are follow-ups.

## Findings (severity-tiered, with UX response)

### Critical — fix before launch

**C1 — `/` (homepage) has no canonical tag.**
`app/page.tsx:16-34` does not export `metadata` — relies on `app/layout.tsx`, which sets `metadataBase` but never declares `alternates.canonical`. Live probe: 0 canonical hits on `/`. Every other route sets it cleanly.
**UX response — Compounds.** When the homepage canonical is absent, every internal nav link (`Nav` and `Footer` link to `/` ~35 times) accumulates link equity into an ambiguous URL set. That fragments the brand-search SERP (people Googling "reveren") into duplicate snippets — a *user-visible* SERP regression.
**Fix:** in `app/page.tsx`, `export const metadata: Metadata = { alternates: { canonical: "/" } }`. Add `description` while you're there.

**C2 — No `app/robots.ts`, no `app/sitemap.ts`.**
Both `/robots.txt` and `/sitemap.xml` return 404. Protocol Step 1 cross-check fails by construction. AI-crawler stance undeclared.
**UX response — Compounds.** When share-aggregators or AI answer engines can't resolve `sitemap.xml`, they fall back to scraping — but the site is gated behind `LAUNCH_MODE=coming-soon` in prod, so any AI crawler hitting today gets only the holding page. `middleware.ts:33-36` already allowlists `/robots.txt` and `/sitemap.xml`, so those files will route correctly *once they exist*.
**Fix:** gate-aware sitemap + robots (see implementation order #3 above). Declare `Allow: GPTBot|ClaudeBot|PerplexityBot|Google-Extended` explicitly.

### High — fix this sprint

**H1 — Per-route Open Graph never overrides layout default.**
Live: `/manifesto`, `/pricing`, `/pods`, `/security`, `/privacy`, `/terms`, `/dpa` all return `og:title = "reveren — One pipeline. Every agent."` and `og:url = "https://reveren.ai"`. Same for `twitter:`.
**UX response — Compounds.** **This is the conversion problem, not the ranking one.** When an engineer pastes `https://reveren.ai/pricing` into Slack to ask "should we pay for this?", the unfurl shows the homepage tagline instead of "$19 Pro / $39 Team". Decision-context is destroyed at the share boundary. For `/manifesto` it's even worse — the pull-quote title is *the* hook.
**Fix:** per page, add `openGraph: { title, description, url: '/<route>' }`. Twitter cards inherit unless overridden — set them on the same money pages.

**H2 — OG image is SVG (`/og/reveren-og-1200x630.svg`, 1.1KB hand-authored vector).**
Twitter, LinkedIn, Slack, Discord, Facebook don't render SVG OG images — fall back to no preview at all. Only Mastodon and a few engineering-leaning clients handle SVG.
**UX response — Compounds, reclassify as Critical for launch.** Engineering Slack + LinkedIn dominate share surfaces for the target audience (engineering leads, devtools founders). Both render zero preview on SVG. This is the highest-impact bug for the social-share funnel.
**Fix:** rasterize to PNG 1200×630 (≤8MB) at the same path, switch `images[0].url`. Keep the SVG as a build-time source.

**H3 — `description` falls back to layout default on `/security`, `/privacy`, `/terms`, `/dpa`.**
All four declare `metadata` but only set `title` and `alternates.canonical` — no `description`. Live HTML returns the generic layout description.
**UX response — Compounds (trust hit).** Someone Googling `"reveren security"` (high-intent buyer due diligence) sees generic marketing copy under a security URL. Brand-trust regression, not just SEO miss.
**Fix:** add 120–160 char descriptive `description` per page. Copy already exists in the body — lift one sentence.

**H4 — `/coming-soon` canonical → `https://reveren.ai` while homepage has no canonical at all.**
Broken canonical chain at launch. Fixing C1 collapses this.
**UX response — Pure-SEO.**

### Medium — track

**M1 — `/pods` has no per-page JSON-LD.**
Marketplace + waitlist + comparison table.
**UX response — Compounds.** Page targets a creator-buyer query class ("author once sell across agents"). Adding `Service` or `ItemList` schema is right; additionally, expose the waitlist endpoint as a *deep-link-able* `#waitlist` anchor so JSON-LD `potentialAction.target` can point to the actual conversion surface (`https://reveren.ai/pods#waitlist`).

**M2 — `/security`, `/privacy`, `/terms`, `/dpa` have no per-page JSON-LD.**
Only layout's `Organization` + `WebSite`.
**UX response — Pure-SEO.** Each is a legal/policy page; `WebPage` + `Article` would be a strict improvement and lets answer engines pull the right summary.

**M3 — FAQPage schema on `/pricing` not asserted to match rendered FAQ accordion text.**
Single source (`lib/faq.ts`) is good, but no test guards drift.
**UX response — Compounds, sharper.** `components/FAQ/FAQ.tsx:81-82` wraps each question in `<summary>` (not `<h3>`), so the page DOM has no heading-level signal for question text even though the schema asserts it. Worse: Google's mismatch penalty also fires when schema asserts a `Question` whose answer wraps in `<details>` that's collapsed by default — answer is in the DOM but visually hidden.
**Fix:** Playwright assertion that `<summary>` text === `faq[i].question`. Consider promoting each `<summary>` to wrap an `<h3>` inside it for both crawler clarity and screen-reader heading navigation.

**M4 — Heading hierarchy on `/` has `<h3>` siblings styled `MuiTypography-body1`.**
`FeatureTeaser`: "Protocols + pipelines", "Protocol generator", "Pods + Marketplace". Functionally correct (h3 after h2) but visual styling is body-1 weight.
**UX response — Compounds.** `components/FeatureTeaser/FeatureTeaser.tsx:100-111` passes `component="h3"` but no `variant` — MUI defaults variant to body1. Real a11y issue: assistive-tech users navigating by headings see three items that look like cards visually but share styling tier with paragraph text. By contrast, `Solution` and `NotThis` correctly render `MuiTypography-h4`. One-component fix.
**Fix:** add `variant="h3"` (or `variant="h4"` to match adjacent sections).

**M5 — Title-length budget — manifesto title could grow past 60 chars.**
**UX response — Tension.** SEO says clip; UX says don't. The site **already does the right thing**: `app/manifesto/page.tsx:24` sets `title: "Manifesto"` (template appends `· reveren` → 17 chars total) for the `<title>`, while the long-form pull-quote lives in `<h1>` + `og:title`. The risk is a future audit "fixing" this back into a clipped SERP-friendly title. **Lock with a test:** assert `metadata.title === "Manifesto"`.

### Low

**L1 — OG image dimensions declared 1200×630 but image is SVG.**
Collapses into H2.

**L2 — `twitter:card` is `summary_large_image` on layout but `summary` on `/coming-soon`.**
Intentional override (`app/coming-soon/page.tsx:31`), harmless.

**L3 — No `hreflang` tags.**
Site is English-only — absence is correct.

## UX-only findings (issues SEO missed)

### 🟥 Critical

- **CLI naming drift between brand and binary.** See top of doc. Launch-blocker.

### 🟧 High

- **`/` above-the-fold doesn't say *what* reveren is.** Eyebrow: "The standards layer for AI agents". H1: "One pipeline. Every agent.". Subline opens: "Stop correcting your AI. Start directing it." A target reader reaches paragraph-end before learning reveren is *a CLI that writes Markdown files into your repo*. Terminal animation in the right column does the explaining — but on mobile (single-column stack) the terminal lands ~600px down, below the entire copy block. **Fix:** reorder mobile stack so terminal sits between subtitle and CTA, OR hoist a one-line "What it is: a CLI (`npx @reveren-ai/core init`) that scaffolds an open Markdown protocol library into your repo" above the install button.
- **`/pricing` price decision below the fold.** `PricingHeader` (`components/PricingHeader/PricingHeader.tsx:13`) renders `2.5rem→3.75rem` headline + 3-line subtitle before the tier grid. On a 13" laptop the $19/$39 cards land at ~750px. **Fix:** collapse `PricingHeader` to single-line h1; pull `TierCards` flush OR render compact price row (Free / $19 / $39 / Custom) inside the header itself.
- **Waitlist conversion path on `/manifesto` is missing.** The page ends with a `TerminalButton` CTA (copy-install) — but `/manifesto` is the strongest emotional surface (readers who consume 2,000 words are the qualified leads). No waitlist CTA. **Fix:** add `WaitlistButton` next to or in place of the `TerminalButton` in the end-of-essay block.
- **`Nav.tsx` waitlist CTA is desktop-only.** Lines 46-72 wrap `WaitlistButton` in `display: { xs: "none", md: "flex" }`. On mobile the only path is through `MobileNav` — 1-extra-tap penalty on the most converting surface, on the device class that's most of marketing-share traffic. **Fix:** persistent mobile waitlist CTA (sticky bottom bar, or pin right of hamburger).
- **`coming-soon` gate destroys deep-share intent.** `middleware.ts` rewrites any path to `/coming-soon` while preserving the URL bar. Today a Slack paste of `https://reveren.ai/pricing` renders generic holding page — and because `/coming-soon` canonicalises to `https://reveren.ai/`, share recipient loses URL fingerprint of "sender wanted me to look at *pricing*". **Third options:** (a) preserve original path in `next-url` and render "you tried /pricing — sign up and we'll email you when it's live"; (b) middleware-level `og:title` rewriting so pricing-link unfurls as "Pricing — coming soon".

### 🟨 Medium

- **`/pricing` sticky head sticks vertically but first-column labels scroll horizontally on mobile.** Once user scrolls right to see Enterprise column, they lose row labels in the leftmost column. **Fix:** `position: sticky; left: 0` on first cell of each row.
- **`Terminal.tsx:36` uses function-form `sx`.** Per project memory (`feedback_mui_rsc_traps.md`) this is a hydration-crash pattern. Component is `"use client"` so it survives, but consistency matters. **Fix:** static-object `sx` + `useTheme()` read if palette-mode-conditional.
- **Hero terminal has both `role="img"` aria-label AND live-text inside.** Screen-reader users hear the aria-label and then *also* the typed terminal text. **Fix:** `aria-hidden="true"` on `<pre>` body; the aria-label already summarises.
- **`AudienceCards` uses no headings inside cards** (`components/AudienceCards/AudienceCards.tsx:52-57`) — eyebrow + body1 per persona, no h3. Heading-nav skip-readers can't traverse the three personas. **Fix:** promote each `.label` to `<h3>`.
- **No focus-visible ring verified on `CopyButton`-as-install-CTA.** `Hero.tsx:91-97` only adds transform-on-hover; check `CopyButton/CopyButton.tsx` separately.

### 🟩 Low

- **`FounderBio` and `OpenFormat` come after `ManifestoTeaser` on homepage** — manifesto teaser is the second-strongest CTA; two more sections push final CTA way down. Consider moving `FounderBio` *before* manifesto teaser as social proof leading into the read.
- **`Footer.tsx:120` line `reveren.ai`** appears as bare text, not link. Either remove or wrap in `<a>`.
- **`<a href="#main" className="rv-skip-link">`** exists in `app/layout.tsx:133` — verify focus styling in `globals.css`.

## Where UX would push back on SEO

- **M5 manifesto title clip.** SEO says clip past 60 chars; UX says the site already does the right thing (short `<title>`, long `<h1>` + `og:title`). Lock with a test, don't "fix" it back.
- **H2 severity placement.** SEO ranks SVG OG image as 🟧 High; UX/conversion reclassifies as 🟥 Critical for launch-readiness. Slack/LinkedIn are where this audience lives.
- **C2 AI-crawler stance "undeclared".** SEO frames silence as ambiguous. UX/business: reveren's open-format/MIT positioning *wants* AI crawlers reading the content (the pitch is reveren is the standards layer the agent reads from). Declare *open*, not just "declared".
- **Implicit SEO frame of "noindex previews."** Preview deploys correctly emit `noindex, nofollow` — but `og:*` should still render. Make sure `next/metadata` doesn't strip OG in tandem with the robots policy.

## Protocol gaps surfaced

- **Review/QA protocols** apparently never flagged that a new `app/<route>/page.tsx` lacks `alternates.canonical` or `description` — those omissions slipped through `/security`, `/privacy`, `/terms`, `/dpa` PRs. Add to review checklist: "every new `page.tsx` exports both `title` and `description`, plus either its own `alternates.canonical` or an explicit comment justifying canonical inheritance from layout (only safe for homepage)."
- **No protocol covers raster-vs-vector OG images.** Add: "OG images shipped to social platforms must be PNG/JPG, not SVG."
- **No protocol covers brand-vs-binary naming drift.** Add a project-level rule that any new copy mentioning the CLI binary must reference `rvr` (or whatever the resolved name becomes), and ship a lint/grep rule in `pnpm pre-push`.
